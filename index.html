Key Prompt — Initial Brief

Replit Agent
Build a full-stack UX research platform called Insightly. Researchers should be able to: create studies, add participants via CSV, send branded invites, run AI-moderated voice interviews, auto-generate transcripts, and get AI insight reports. Stack: React 19 + TypeScript + Tailwind v4 (frontend), Node.js + Express 5 + Drizzle ORM + PostgreSQL (backend), pnpm monorepo with shared Zod schemas. For AI: Anthropic Claude API for dynamic follow-up questions and insight generation. Google Cloud TTS/STT for voice. Nodemailer for emails.

Honest Reflection

Challenges & Learnings

Three real friction points hit during the build — and how each was solved.

Voice sync issues

STT would activate before TTS finished speaking, creating feedback loops between the AI voice and the mic.

Added a speaking state flag that blocks STT until TTS playback fully ends.

Claude repeating questions

Early versions would repeat the original question verbatim as a "follow-up" when participants gave short answers.

Passed full conversation history in every API call and explicitly instructed Claude never to repeat asked questions.

Claude API cost at scale

Multiple API calls per session (questions + follow-ups + insights) adds up fast at Sonnet pricing.

Implemented token budget per session and truncated older context to keep costs predictable.

Mobile mic permissions

iOS Safari and Chrome Android handled microphone permission prompts differently — breaking the interview flow on mobile.

Added browser-specific permission prompting and rebuilt the interview UI mobile-first.
